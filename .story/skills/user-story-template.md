# Skill: User Story Template

## Purpose

Produce clear, consistent stories that are ready for refinement.

## Template (required output format)

Title: <Concise summary>

As a <persona/role>,
I want <capability/feature>,
so that <benefit/outcome>.

Context & Constraints:

- <assumptions>
- <non-functional constraints: performance/security/accessibility>

Acceptance Criteria:

- AC-1: <Given> <When> <Then> (must be testable)
- AC-2: ...
  (Use Gherkin for complex flows)

Definition of Done (summary, link to full DoD):

- Code written, reviewed, and merged
- Automated tests added
- Docs/README updated
- E2E/Integration tests (if applicable)
- Accessibility checks passed

Effort estimate: <T-shirt / story points>
Priority: <P0/P1/P2> (reason)
Notes / UX:

- <wireframe/behaviour notes>
