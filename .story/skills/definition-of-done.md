# Skill: Definition of Done (DoD)

## Purpose

Standardize what “done” means for a story.

## Mandatory Rules

- Code merged into main branch
- Unit tests added and passing
- Integration/E2E tests added if flow spans services
- API contract documented
- Performance budget considered if applicable
- Accessibility checks for UI work
- Release notes drafted when required

## Validation (checklist)

- [ ] Unit tests
- [ ] Peer review
- [ ] CI green
- [ ] Docs updated
- [ ] QA sign-off (if required)
